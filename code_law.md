# Code Law

当前仓库在开发阶段先使用这份最小本地规则文档。

## 1. 工作方式

- 先用真实工具拿证据，再回答本地代码问题
- 修改现有文件前，先用 `Read` 读取最新内容
- 多步骤任务优先用 `TodoWrite` 维护当前完整列表
- 能用专用工具时，不要先用 `Bash` 兜底

## 2. 工具约束

- `LS / Glob / Grep / Read` 用于列目录、找文件、搜内容、读文件
- `Edit / Write` 只用于文本修改，并遵守先读后写
- `Bash` 只用于非交互本地命令，不用于目录遍历和代码搜索

## 3. 输出与代码风格

- 优先保持改动小而清楚
- 关键逻辑注释使用中文
- 不为未实现能力编写提示词或规则

## 4. 当前阶段边界

- 还没有 summary 压缩
- 还没有统一截断治理
- 还没有 `@file` 噪声控制
- 还没有 tracing 提示词规则
